# 赛区与战队设计

## 四大赛区

游戏固定4个赛区，每个赛区14支战队：

| 赛区代码 | 赛区名称 | 地区 | 特点 |
|---------|---------|------|------|
| LPL | 中国职业联赛 | 中国大陆 | 选手赛区偏好最高 |
| LCK | 韩国职业联赛 | 韩国 | 选手赛区偏好较高 |
| LEC | 欧洲职业联赛 | 欧洲 | 选手赛区偏好中等 |
| LCS | 北美职业联赛 | 北美 | 选手赛区偏好较低 |

## 战队配置

### 每赛区战队数量

- 每赛区固定 **14支战队**
- 总计 **56支战队**
- 战队名称可修改，但数量固定

### 战队属性

| 属性 | 说明 |
|------|------|
| `id` | 唯一标识符 |
| `name` | 战队全称 |
| `short_name` | 战队简称 |
| `region_id` | 所属赛区 |
| `power_rating` | 战力值（根据首发阵容计算） |
| `balance` | 资金余额 |
| `annual_points` | 年度积分 |

### 阵容配置

每支战队必须有5个位置的选手：

| 位置 | 英文 | 说明 |
|------|------|------|
| 上单 | TOP | 上路选手 |
| 打野 | JUG (Jungle) | 打野选手 |
| 中单 | MID | 中路选手 |
| ADC | ADC | 下路射手 |
| 辅助 | SUP (Support) | 辅助选手 |

**首发规则**:
- 每队每位置最多1名首发
- 系统自动选择能力值最高的选手作为首发
- 能力值相同时，选择年龄较大的选手

## 赛区特性

### 赛区偏好值

选手有 `home_region_id`（出生赛区）和 `region_loyalty`（赛区偏好值）：

| 出生赛区 | 偏好值范围 | 说明 |
|---------|-----------|------|
| LPL | 75-90 | 中国选手最不愿意外流 |
| LCK | 55-75 | 韩国选手较不愿意外流 |
| LEC | 45-65 | 欧洲选手中立 |
| LCS | 40-60 | 北美选手较愿意外流 |

**跨赛区转会意愿计算**:

```
跨赛区转会意愿 = 基础意愿 × (100 - region_loyalty) / 100
```

### 赛区声望

赛区整体实力影响选手转会意愿：

- LPL/LCK 选手更愿意留在本赛区
- LCS 选手更容易被其他赛区挖走
- 国际赛成绩影响赛区整体声望

## 战队管理界面

### 战队列表

以表格展示所有战队：

- **基础信息**: 战队名称、所属赛区
- **数据概览**: 总场次、胜场、胜率、年度积分
- **筛选功能**: 按赛区、胜率区间、战力值区间筛选

### 战队详情页

- **基础信息**: 名称、赛区、战力值
- **阵容**: 首发和替补选手
- **财务**: 资金余额、薪资支出、收入来源
- **荣誉**: 历史荣誉记录
- **历史**: 各赛季战绩

### 编辑功能

- 修改战队名称
- 调整首发阵容（如果有替补）
- 查看选手合同状态

## 战力值计算

战队战力值 = 5名首发选手能力值的平均值

```
team_power = (TOP.ability + JUG.ability + MID.ability +
              ADC.ability + SUP.ability) / 5
```

战力值在每赛季开始时更新，赛季中保持不变。
